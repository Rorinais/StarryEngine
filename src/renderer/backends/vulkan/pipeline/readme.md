# 接口
- IPipeline
- IPipelineStateComponent

# 编码流程
1. 实现 IPipelineStateComponent 接口的各个组件类，用于描述管线状态的各个方面（如着色器、输入布局、混合状态等）。
2. 实现 IPipeline 接口的，用于创建各种类型的管线（图形管线、计算管线等）。
3. 用ComponentManager注册各个管线状态组件类，以便在创建管线时能够动态组合所需的状态组件。
4. 实现 pipelineBuilder 模块，用于根据传入的状态组件创建完整的管线对象。ComponentManager可以选择将不同组件组合成不同类型的管线。
5. 实现管线缓存机制，以提高管线创建的性能。可以使用哈希表或其他数据结构来存储和检索已创建的管线。
6. 实现管线管理器，用于管理管线的生命周期，包括创建、销毁和重用管线对象。
7. 将管线管理器集成到全局唯一的资源管理器中
